css:box-shadow

2020-07-03

ie8以上

box-shadow 怎么产生的

  1. 首先在元素下面生成一个一样的大小的，位置重叠的，颜色等同于《color》的div
  2. 四边向外或者向内做高斯模糊，在blur-radius长度内做完模糊，类似正态分布，只是边就算是坐标0点
  3. 根据xy-offset偏移量移动
  4. 删除和真实div重合部分

box-shadow 的属性较多

  • ‘inset’ 有则阴影在边框内部，阴影会在边框之内 (即使是透明边框）、背景之上、内容之下
  • x-offset 向右偏移量
  • y-offset 向下偏移量
  • blur-radius 模糊半径。值越大，模糊面积越大。 默认0，此时阴影边缘是锐利的，不能是负的。（高斯模糊）
  • spread-radius 那个根据真实的div产生的虚拟元素的大小扩展半径
  • color

内阴影，四边阴影，双边阴影

  • 四边阴影

  • 相临边阴影

单边阴影

这个也好懂了。

  1. 首先四边模糊半径10px；
  2. 然后向下偏移10px；
  3. 四边阴影长度分别是： 上右下左： 0 10px 20px 10px;
  4. 然后-10px的阴影扩展长度；(各减去10)
  5. 四边阴影长度分别是： 上右下左： -10px 0px 10px 0px;

css:BFC

2020-07-03

Block Formatting Content

定义

块级格式化上下文

首先明确几个定义

  • 普通流
    普通流元素，行内元素横向排列，超出换行。块级元素独占一行。上至下排列。

  • 浮动
    在浮动布局中，元素首先在正常文档流位置，然后根据浮动方向，尽可能的向这个方向移动。渲染效果和文字环绕效果类似。

  • 绝对定位
    绝对定位的元素完全脱离了文档流，不会占用普通文档流的位置，根据定位信息确定位置，在另外的层级。

    BFC属于普通流元素

    它可以被视作一个完全和外界隔离的独立容器，不会对外界产生影响

触发元素的BFC

  • root元素

  • display: inline-block; table-cell; flex;

  • float 除了none不设置之外的

  • position: fixed; absolute; 绝对定位元素

  • overflow: visible 之外的值 auto scroll hidden

    可以知道，这些元素的内部无论怎么变化折腾，都不会对外界元素产生影响。
    也就可以理解为什么overflow:visible 会被排除在外了。

BFC内部

root元素都是BFC，可以说整个DOM文档流都是BFC内部元素。

刚刚说了BFC不会对外部元素影响。

那么它内部的非BFC元素还是会互相影响的。

互相影响会出现的问题

  • 上下边距margint融合问题
  • 浮动（怎么清除浮动）
  • 占用其他元素内容区域（float）

BFC可以做什么

通过给元素设置成BFC，解决问题：

  • 上下边距融合问题
    根据BFC特性，只要把其中一个元素设置成BFC（增加overflow:hidden），它就不会对外界元素产生影响。

  • 清除浮动
    给浮动元素设置成BFC，即使它子元素只有浮动元素。也会被撑起来。因为BFC的内部元素不应该超出自己的元素内部。

  • 两栏布局
    左侧元素float。右侧元素的一部分会被浮动元素占据，形成环绕效果。

    给右侧元素设置成BFC，可以实现两栏布局。

    左右都是BFC不会互相影响。

js基础 --原型相关方法，Object，Reflect

2020-05-02

原型相关的方法们

  • isPrototypeOf()
    用于测试一个对象是否存在于另一个对象的原型链上。

  • Object.getPrototypeOf()
    返回原型对象
    给定对象的原型。如果没有继承属性，则返回 null 。

  • hasOwnProperty()
    方法会返回一个布尔值，指示对象自身属性中是否具有指定的属性（也就是，是否有指定的键）。

  • Object.keys()
    for…in 也是只遍历可枚举属性
    方法会返回一个由一个给定对象的自身可枚举属性组成的数组，数组中属性名的排列顺序和正常循环遍历该对象时返回的顺序一致 。

    只会显示出可枚举的属性

    如果需要把不可枚举的也列出来，就用到下面的方法了

  • Object.getOwnPropertyName()
    方法返回一个由指定对象的所有自身属性的属性名（包括不可枚举属性但不包括Symbol值作为名称的属性）组成的数组

  • Reflect和原型相关的方法：

    1. Reflect.getPrototypeOf()
    2. Reflect.setPrototypeOf()
    3. Reflect.ownKeys()
    4. Reflect.has()

属性描述相关方法

  • Object.getOwnPropertyDescriptor(obj, prop)
    方法返回指定对象上一个自有属性对应的属性描述符。（自有属性指的是直接赋予该对象的属性，不需要从原型链上进行查找的属性）
    有则返回该属性的属性描述符，无则返回undefined
    属性描述：
    • value // 属性的值
    • writable // 是否可改变， Boolean
    • get // 该属性访问器函数，可以为undefined
    • set // 该属性的设置器函数，可以为undefined
    • enumerable // 该属性可以被枚举，Object。keys() 可以找出来
    • configurable // 该属性的属性描述是否可以被更改或者删除
  • Object.getOwnPropertyDescriptors(obj)
    获取一个对象自身所有属性的属性描述
  • Object.getOwnPropertySymbols()
    顾名思义
  • a.propertyIsEnumerable(‘b’)
    判断a的属性b是否可枚举，返回值Boolean
  • Object.defineProperties(object, props)
    直接在一个对象上定义新的属性s或修改现有属性s，并返回该对象。
  • Object.defineProperty(prop)
    会直接在一个对象上定义一个新属性，或者修改一个对象的现有属性，并返回此对象。
  • Reflect中与属性描述相关的方法
    1. Reflect.defineProperty()
    2. Reflect.deleteProperty()
    3. Reflect.getOwnPropertyDescriptor()
    4. Reflect.isExtensible() // 是否可扩展
    5. Reflect.preventExtensions() // 禁止扩展

es6的继承

es6通过extends关键字进行继承

首先定义一个父类Father

需要注意的是

  • constructor函数

    它代表对实例对象本身写属性方法，class的属性方法相当于写在了构造函数的原型对象上。
它一定存在，没有会默认生成一个

  • super关键字

    1. 在contructor中做函数使用的时候，super()调用会生成一个空对象，作为context来调用父类的constructor，返回this对象，作为子类constructor的context继续调用构造函数。
    2. 作为对象，它就代指向父类

总结

ES5 的继承，实质是先创造子类的实例对象this，然后再将父类的方法添加到this上面（Parent.apply(this)）。ES6 的继承机制完全不同，实质是先将父类实例对象的属性和方法，加到this上面（所以必须先调用super方法），然后再用子类的构造函数修改this。
